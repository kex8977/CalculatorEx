사칙연산부터 날짜계산, 길이, 거리, 무게 등 다양한 기능을 제공했고, 메모리 기능도 있었다.

그래도 오늘 목표는 사칙연산이 가능한 계산기를 만들어 보는 것

나름대로 신경쓴다고 Toolbar를 먼저 배치하고
그 사이에 UIBarButton들을 넣어줬다.

또한 1라인에 4개의 버튼이 적당할 것 같아
Flexible Space Bar Button Item 도 넣었지만...

디자인은 한 번 손봐야할 것 같다...ㅎ



이렇게 최소한의 기능만 구현을 목표로 했는데도 버튼이 16개 였다.
각각 버튼들의 이름을 붙혀주고, 해당 버튼이 눌릴 때 일어날 일들을 코딩을 해줬다.

<img width="469" alt="스크린샷 2024-03-06 오후 9 52 51" src="https://github.com/kex8977/---iOS-3-/assets/46492883/72ac8691-37ce-4169-ac51-44203354ea8a">


여기까지 어려움은 없었으나
= 버튼을 누르는 순간에 에러가 나고 제대로 된 연산이 이루어지지 않아 어려움이 있었다

사유는 string 값이 옵셔널 하다는 것
즉, historyBox.text 와 inputBox.text 값이 string이 아닌 nil 이 올 수도 있다는 의미였다.

물론 나의 두뇌로는 당연하게 string을 저장하였으니 nil 값이 저장될 일이 없으나
컴파일러의 입장에서 생각하면 그딴 건 사람 입장이라는 거다

<img width="1071" alt="스크린샷 2024-03-06 오후 6 46 44" src="https://github.com/kex8977/---iOS-3-/assets/46492883/d87f430d-71fd-4c56-8b16-b7e523097595">

분명히 며칠 전 옵셔널의 정의가 무엇인지,
그것을 활용하여 옵셔널 바인딩, 체이닝도 공부를 했으나 좀처럼 이해가 되지 않았다.

결국 튜터님에게 찾아가서 상황을 설명하고 조언을 구했고,
2가지 작업을 해야한다라고 답변해주셨다.

1. 옵셔널을 해제하기
2. 해제된 값이 숫자가 확실한지

<img width="454" alt="스크린샷 2024-03-06 오후 9 53 21" src="https://github.com/kex8977/---iOS-3-/assets/46492883/68c2e586-4b04-450b-9bdb-32bf1b633321">

많은 검색으로 다른 사람들이 찾아보았지만
단순한 값 정의 후 print문을 반복하여 결과를 확인하는 등의 글이 많이 있었다.
또한 왜 옵셔널 값을 잘 다루어야 하는지도 알았으나

나의 현 상황의 오류를 해결해주지는 않았다.

옵셔널 바인딩에는 여러가지 기법들이 있다고는 하지만,
내가 사용한 것은 강제로 언래핑하여 값을 저장해주는 방법을 채택했다.

이 부분을 if let 이나 guard else 구문으로 
혹은 다른 방법으로 대체할 수 있을까?

과연 그렇게 한다면 좀더 깔끔하고 안정성 있는 프로그램이 된다면 공부해봐야겠다
